Haskell Chess Engine AI and Terminal Interface Demo

This demo page showcases the substantial chess engine interface and AI opponent functionality built as part of a group project in Haskell. The project features a fully functional terminal application, complete local opponent matchups, importing board test cases, depth toggles, verbosity toggles, and more. We also implemented early exit conditions to improve the optimization of the number of nodes expanded by the program to gain a significant efficiency boost.

The source code, boards, and test cases can be found at Haskell Chess.

We begin by launching the application:

After starting the application, we are greeted with the available flags

We can make use of the '-t' flag to toggle two player local play, with an inverting board for each player. There is an option to select a non-traditional starting board which can be loaded inside the aptly named "txtcases" using txt files and the format provided.

A standard start to a two player interactive game.

By using the '-i' flag, we can launch an interactive game. We can specify a specific board to begin interactive play from, as well as a depth limit if we want to run a deeper or shallower calculation. After a few seconds, the AI makes the move, and the board is returned with the move input reflected on the ASCII.

An opening game AI move, with default no flags.

An late game AI Move, using one of the "txt" cases.

We can run the AI on arbitrary positions by using or writing our own "txtcase." In this case, the checkmate is in the depth range, so the algorithm finds the right move. We specified the depth as 6 using "-d 6" rather than the standard 5, and this can be used with the "-v" tags for visual output as well.

An estimate calculation for a checkmate in two.

In addition to our estimating solver, we included an absolute solver as well, using the '-w' flag. This one calculates an uncapped amount of time until it finds a checkmating move. This one is recommended only for games where a known checkmate exists, and serves no utilitarian purpose for regular games because of its slow calculation times.

An absolutely calculated checkmate in one. This can be paired with the '-v' verbose tags to produce the visual output as well as rate the position for side whose turn it is.
